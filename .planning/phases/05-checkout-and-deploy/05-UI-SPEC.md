# Phase 5 — UI Design Contract

> Visual and interaction contract for the checkout flow, confirmation page, and email templates. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components) |
| Icon library | Inline SVG (Heroicons outlines, manually inlined) |
| Font | Geist Sans (`--font-geist-sans`) via `next/font/google` |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `p-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing, page horizontal padding (`px-4`, `gap-4`) |
| lg | 24px | Section padding, form field gaps (`gap-6`, `py-6`) |
| xl | 32px | Layout gaps between major sections (`gap-8`) |
| 2xl | 48px | Page-level vertical padding for confirmation hero (`py-12`) |
| 3xl | 64px | Top/bottom page breathing room (`py-16`) |

Exceptions: Touch target minimum 44px height for the submit button on mobile (using `py-3` on a text-sm button achieves this).

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 500 (medium) | 1.5 | `text-sm font-medium` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

Source: Existing codebase patterns. Cart page uses `text-xl font-semibold` for "Your Cart"; header uses `text-lg font-bold`. Display size (24px) is new for the confirmation page "Order placed!" heading. Body and label sizes match existing `text-sm` / `text-sm font-medium` usage across all components.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `bg-white` (dark: `#0a0a0a` / `bg-zinc-950`) | Page background, form background |
| Secondary (30%) | `#fafafa` / `bg-zinc-50` (dark: `#18181b` / `bg-zinc-900`) | Order summary item rows, card container backgrounds |
| Accent (10%) | `#4f46e5` / `bg-accent` (dark: same) | Submit button, "Edit cart" link text, confirmation checkmark icon |
| Accent hover | `#4338ca` / `bg-accent-hover` | Submit button hover state |
| Accent light | `#eef2ff` / `bg-accent-light` | Confirmation page checkmark background circle |
| Destructive | `#ef4444` / `text-red-500` | Error message text only |
| Warning | `#d97706` / `text-amber-600` | Not used in this phase |

Accent reserved for:
- "Place order" submit button (`bg-accent text-white`)
- "Edit cart" link text (`text-accent`)
- Confirmation checkmark icon fill (`text-accent`)
- Confirmation checkmark background circle (`bg-accent-light`)
- "Browse more cards" link on confirmation page (`bg-accent text-white`)
- Email template header color (`#4f46e5` inline style)

## Layout Contract

### Checkout Page (`/checkout`)

```
+------------------------------------------------------+
| Header (reused)                                       |
+------------------------------------------------------+
|  max-w-3xl mx-auto px-4 pt-6 pb-24                  |
|                                                       |
|  [Checkout]  (text-xl font-semibold)                 |
|                                                       |
|  MOBILE (< 768px): Single column, stacked            |
|  +-------------------------------------------------+ |
|  | Order summary                     [Edit cart ->] | |
|  | +---------------------------------------------+ | |
|  | | [thumb] Card Name     x2    $4.50            | | |
|  | | [thumb] Card Name     x1    $2.00            | | |
|  | +---------------------------------------------+ | |
|  | Total                             $6.50         | |
|  +-------------------------------------------------+ |
|                                                       |
|  +-------------------------------------------------+ |
|  | Your details                                     | |
|  | Name      [________________]                     | |
|  | Email     [________________]                     | |
|  | Message   [________________]  (optional)         | |
|  |                                                  | |
|  | [  error message if any  ]                       | |
|  | [        Place order         ]                   | |
|  +-------------------------------------------------+ |
|                                                       |
|  DESKTOP (>= 768px): Side-by-side                    |
|  +------------------------+ +----------------------+ |
|  | Your details           | | Order summary        | |
|  | Name  [__________]     | | [Edit cart ->]       | |
|  | Email [__________]     | | [thumb] Card  x2 $4 | |
|  | Msg   [__________]     | | [thumb] Card  x1 $2 | |
|  |                        | | Total        $6.50   | |
|  | [error if any]         | |                      | |
|  | [  Place order  ]      | |                      | |
|  +------------------------+ +----------------------+ |
+------------------------------------------------------+
```

**Layout rationale (D-01, Claude's discretion):** Mobile-first stacked layout with order summary on top (user reviews order before filling form). On desktop (md breakpoint, 768px), form on left (2/3 width) and order summary on right (1/3 width) as a sticky sidebar. This follows the existing `max-w-3xl` pattern from the cart page.

### Order Summary Row

Each order line item renders as:
- 48x67px thumbnail (same dimensions as `cart-item.tsx`)
- Card name (text-sm font-medium, truncate)
- Set name (text-xs text-zinc-400, truncate)
- Quantity badge: "x{N}" (text-sm text-zinc-500)
- Line total: "${price * qty}" or "N/A" (text-sm, right-aligned)
- Row background: `bg-zinc-50 dark:bg-zinc-900 rounded-lg p-3`
- Row gap: `gap-3` (12px)
- No quantity controls (D-04: read-only)
- No remove button (D-04: read-only)

## Interaction States

### Form Fields

| State | Visual |
|-------|--------|
| Default | `border border-zinc-300 dark:border-zinc-600 rounded-md px-3 py-2 bg-transparent text-sm` |
| Focus | `outline-none ring-2 ring-accent/50 border-accent` |
| Error (HTML validation) | Browser-native validation bubble (no custom error styling) |
| Disabled (during submit) | `opacity-50 cursor-not-allowed` |

### Submit Button

| State | Visual | Copy |
|-------|--------|------|
| Idle | `bg-accent text-white rounded-md px-5 py-3 text-sm font-medium w-full` | "Place order" |
| Hover | `bg-accent-hover` | "Place order" |
| Submitting | `opacity-70 cursor-not-allowed` + no hover effect | "Placing order..." |
| Disabled (empty cart) | `opacity-30 cursor-not-allowed bg-zinc-300 text-zinc-500` | "Place order" |

### Error Display

| State | Visual |
|-------|--------|
| Error visible | `bg-red-50 dark:bg-red-950/20 border border-red-200 dark:border-red-800 rounded-md p-3 text-sm text-red-600 dark:text-red-400` above submit button |
| Error with retry | Error message + "Try again" button styled same as primary CTA |

### Page Loading (Hydration Guard)

Same skeleton pattern as `cart-page-client.tsx`:
- Pulsing placeholder for heading: `h-8 w-32 bg-zinc-100 dark:bg-zinc-800 rounded animate-pulse`
- Pulsing placeholder for order rows: 3x `h-20 bg-zinc-50 dark:bg-zinc-900 rounded-lg animate-pulse`
- Pulsing placeholder for form fields: 3x `h-10 bg-zinc-100 dark:bg-zinc-800 rounded animate-pulse`

## Responsive Breakpoints

| Breakpoint | Checkout Layout | Confirmation Layout |
|------------|----------------|---------------------|
| < 768px (mobile) | Single column: order summary stacked above form | Centered, full width within `max-w-md` |
| >= 768px (md) | Side-by-side: form (2/3) + sticky order summary (1/3) | Same centered layout (already narrow) |

Container widths:
- Checkout: `max-w-3xl` (matches cart page)
- Confirmation: `max-w-md` (narrow centered content)
- Both: `mx-auto px-4`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has a visible `<label>` element with `htmlFor` matching input `id` |
| Required fields | `required` attribute on name and email inputs; `aria-required="true"` |
| Email validation | `type="email"` on email input for browser-native validation |
| Submit feedback | Button text changes to "Placing order..." during submission; `aria-busy="true"` on form |
| Error announcement | Error container has `role="alert"` for screen reader announcement |
| Focus management | On error, focus moves to error message container |
| Keyboard navigation | All interactive elements reachable via Tab; form submits on Enter |
| Color contrast | Accent (#4f46e5) on white meets WCAG AA (contrast ratio 5.9:1); white on accent meets AA (same) |
