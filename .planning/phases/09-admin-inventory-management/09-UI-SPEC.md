## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline cell padding, badge internal padding |
| sm | 8px | Table cell vertical padding, compact element spacing, input padding-y |
| sm+ | 12px | Action bar padding-y (`py-3`), table header padding |
| md | 16px | Container horizontal padding (`px-4`), table cell horizontal padding, filter bar gaps |
| lg | 24px | Content area top padding (`pt-6`), section gaps between toolbar and table |
| xl | 32px | Major section gaps |
| 2xl | 48px | Page-level vertical spacing |

Exceptions: `sm+` (12px) matches established codebase pattern from admin header `py-3`.

Source: Inherited from Phase 8 UI-SPEC. Matches existing admin layout (`px-4`, `py-3`, `pt-6`).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption | 12px (`text-xs`) | 400 (regular) | 1.5 |
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 600 (semibold) | 1.5 |
| Heading | 20px (`text-xl`) | 700 (bold) | 1.2 |

Font weights used: 400 (regular) and 700 (bold), with 600 (semibold) for table column headers only.

Role usage:
- **Caption** (12px regular): Pagination info ("Showing 1-50 of 312"), low stock badge text, result counts, condition abbreviations in table
- **Body** (14px regular): Table cell text (card name, set, price, quantity), search input text, filter dropdown text, inline edit input text
- **Label** (14px semibold): Table column headers (Name, Set, Price, Condition, Qty, Actions), filter labels, "Export CSV" button text
- **Heading** (20px bold): Page heading ("Inventory")

Source: Inherited from Phase 8 UI-SPEC. Added Label role (14px semibold) for table column headers -- needed for data table visual hierarchy.

## Page Layouts

### Admin Inventory Page (`/admin`)

Structure: Renders inside existing admin layout (header with "Viki Admin" badge and sign-out). Replaces the current placeholder page.

```
+----------------------------------------------------------+
| Viki  [Admin]                              [Sign out]    |  <- existing admin header
+----------------------------------------------------------+
|                                                          |
|  Inventory                                               |  <- text-xl font-bold
|                                                          |
|  +------------------------------------------------------+|
|  | [Search by name...]  [All sets v] [All conditions v] ||  <- action bar
|  |                                        [Export CSV]   ||
|  +------------------------------------------------------+|
|                                                          |
|  +------------------------------------------------------+|
|  | Img | Name    ^| Set    | Price ^| Cond | Qty ^| Act ||  <- table header (zinc-50 bg)
|  +------------------------------------------------------+|
|  |[img]| Card A   | SLD    | $2.50  | NM   |  3   | [x]||  <- normal row
|  +------------------------------------------------------+|
|  |[img]| Card B   | MH2    | $0.75  | LP   |  1   | [x]||  <- low stock (amber left border)
|  +------------------------------------------------------+|
|  |[img]| Card C   | SLD    | [$1.00]| [NM] | [2]  | [x]||  <- editing state (cells become inputs)
|  +------------------------------------------------------+|

|  +------------------------------------------------------+|

|                                                          |
+----------------------------------------------------------+
```

- Container: inherits `max-w-7xl mx-auto px-4 pt-6` from admin layout
- Page heading: `text-xl font-bold` with `mb-6` (24px gap to action bar)

### Action Bar

```
+----------------------------------------------------------+
| [Search by name...]  [All sets v]  [All conditions v]    |
|                                            [Export CSV]  |
+----------------------------------------------------------+
```

- Bar: `flex flex-wrap items-center gap-3 mb-6`
- Search input: `w-64 rounded-md border border-zinc-300 dark:border-zinc-700 bg-transparent px-3 py-1.5 text-sm focus:outline-none focus:ring-2 focus:ring-accent`
- Filter dropdowns: native `<select>` elements -- `rounded-md border border-zinc-300 dark:border-zinc-700 bg-transparent px-3 py-1.5 text-sm` (per D-10: admin-specific, not storefront components)
- Export button: `ml-auto px-4 py-1.5 text-sm font-semibold rounded-md bg-accent text-white hover:bg-accent-hover transition-colors` (per D-12: top-right of action bar)

### Data Table

- Table wrapper: `w-full overflow-x-auto` for horizontal scroll on smaller viewports
- Table element: `w-full text-sm`
- Header row: `bg-zinc-50 dark:bg-zinc-800 text-left`
- Header cells: `px-4 py-3 text-sm font-semibold text-zinc-600 dark:text-zinc-400`
- Sortable headers: `cursor-pointer hover:text-zinc-900 dark:hover:text-zinc-200` with sort arrow indicator (up/down SVG arrow, 12px, inline after text)
- Body rows: `border-b border-zinc-100 dark:border-zinc-800`
- Body cells: `px-4 py-2 text-sm`
- Row hover: `hover:bg-zinc-50 dark:hover:bg-zinc-900/50`

### Column Specifications

| Column | Width | Content | Sortable |
|--------|-------|---------|----------|
| Image | 48px fixed | Card thumbnail 32x45px (`rounded`, `object-cover`) | No |
| Name | flex-1 (min 160px) | Card name, `truncate` on overflow | Yes (A-Z / Z-A) |
| Set | 80px | Set code (uppercased, e.g. "SLD") | No |
| Price | 80px | `$X.XX` or `N/A` for null prices | Yes (asc / desc) |
| Condition | 64px | Abbreviation (NM, LP, MP, HP, DMG) | No |
| Quantity | 56px | Integer | Yes (asc / desc) |
| Actions | 48px | Delete icon button (trash) | No |

### Low Stock Highlight (D-05, INV-06)

Cards with `quantity === 1` receive:
- Left border: `border-l-2 border-amber-500` on the table row
- Badge: `inline-flex items-center px-1.5 py-0.5 text-[10px] font-semibold rounded bg-amber-100 text-amber-700 dark:bg-amber-900/30 dark:text-amber-400` reading "Low" next to the quantity value

This dual treatment ensures low stock stands out at a glance in both full-width and scrolled views.

### Inline Edit Cells (D-06, D-07)

**Click-to-edit behavior:** Clicking a Price, Condition, or Quantity cell transitions that single cell from display mode to edit mode.

**Price cell (edit mode):**
- Input: `type="number" step="0.01" min="0"` styled `w-20 rounded border border-accent bg-white dark:bg-zinc-900 px-2 py-1 text-sm focus:outline-none focus:ring-2 focus:ring-accent`
- Shows `$` prefix outside the input

**Condition cell (edit mode):**
- Native `<select>` element with options: NM, LP, MP, HP, DMG (per D-08)
- Styled: `rounded border border-accent bg-white dark:bg-zinc-900 px-2 py-1 text-sm focus:outline-none focus:ring-2 focus:ring-accent`

**Quantity cell (edit mode):**
- Input: `type="number" min="0" step="1"` styled identically to price input but `w-16`
- Setting quantity to 0 is allowed (card remains in inventory with zero stock)

**Save trigger:** Enter key or blur (D-07). No save button.

**Success indicator:**
- Cell briefly flashes `bg-accent-light` (indigo-50) for 1.5 seconds
- Small green checkmark icon (16px) appears next to the cell value, fading out over 1.5 seconds
- Uses CSS transition: `transition-colors duration-300`

**Error indicator:**
- Cell border turns `border-red-500` for 2 seconds
- Toast notification at bottom of page: `fixed bottom-4 left-1/2 -translate-x-1/2 bg-red-600 text-white text-sm px-4 py-2 rounded-md shadow-lg` showing "Failed to save changes. Try again." Auto-dismisses after 3 seconds.

### Delete Confirmation Row (D-14)

When the admin clicks the delete (trash) icon:
1. The entire row content is replaced with the confirmation state
2. Row background changes to `bg-red-50 dark:bg-red-950/20`
3. Row content: `Delete {card name}?` text (14px regular) with `Confirm` and `Cancel` buttons

**Confirm button:** `px-3 py-1 text-sm font-semibold rounded-md bg-red-600 text-white hover:bg-red-700 transition-colors`
**Cancel button:** `px-3 py-1 text-sm font-semibold rounded-md border border-zinc-300 dark:border-zinc-600 hover:bg-zinc-100 dark:hover:bg-zinc-800 transition-colors`

Clicking Cancel restores the original row. Clicking Confirm sends the delete request, shows a brief loading state (confirm button text changes to "Deleting..." with `opacity-70`), then removes the row from the table.

### Pagination Controls (D-03)

```
+----------------------------------------------------------+
| Showing 1-50 of 312 cards      [< Prev] 1 2 3 ... [Next>]|
+----------------------------------------------------------+
```

- Container: `flex items-center justify-between mt-4 py-3`
- Result info: `text-xs text-zinc-500 dark:text-zinc-400`
- Page buttons: `min-w-[32px] h-8 flex items-center justify-center rounded-md text-sm`
  - Default page: `text-zinc-600 dark:text-zinc-400 hover:bg-zinc-100 dark:hover:bg-zinc-800`
  - Active page: `bg-accent text-white font-semibold`
  - Disabled (prev on page 1, next on last): `opacity-30 cursor-not-allowed`
- Previous/Next: `px-3 h-8 rounded-md border border-zinc-300 dark:border-zinc-600 text-sm hover:bg-zinc-100 dark:hover:bg-zinc-800`
- Show max 5 page numbers with ellipsis for ranges beyond that

### CSV Export (D-12, D-13)

Clicking "Export CSV" triggers an immediate download. No modal, no preview.
- Button shows brief loading state: text changes to "Exporting..." with `opacity-70 cursor-not-allowed`
- On success: browser downloads file named `viki-inventory-{YYYY-MM-DD}.csv`
- On error: same bottom toast pattern as edit errors: "Failed to export CSV. Try again."
- Export always includes full inventory regardless of current filters (D-13)

## Responsive Behavior

Per REQUIREMENTS.md out-of-scope: "Mobile-optimized admin panel -- Admin tasks are desktop workflows." No mobile breakpoints are designed. The table uses `overflow-x-auto` so it remains scrollable on narrower screens, but no layout reflow is required.

Minimum comfortable width: 768px (md breakpoint). Below that, the table scrolls horizontally within its container.

## Accessibility Notes

- Table must use semantic `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>` elements
- Sortable column headers must use `<button>` inside `<th>` with `aria-sort="ascending"` / `"descending"` / `"none"`
- Inline edit inputs must have `aria-label` describing the field (e.g., `aria-label="Price for {card name}"`)
- Condition select must have `aria-label="Condition for {card name}"`
- Delete button must have `aria-label="Delete {card name}"`
- Delete confirmation must use `role="alertdialog"` with `aria-label="Confirm deletion of {card name}"`
- Error toasts must use `role="alert"` for screen reader announcement
- Success indicators should use `aria-live="polite"` region
- Pagination buttons must have `aria-label="Go to page {n}"`, `aria-current="page"` on active page
- Search input must have `aria-label="Search cards by name"`
- Low stock badge must be visible to screen readers (not just color -- the "Low" text badge serves this)
- Focus must return to the table after a successful delete (focus the row that moves into the deleted row's position)
