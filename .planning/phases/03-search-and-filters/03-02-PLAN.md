**src/components/multi-select.tsx** ('use client'):
1. This is a REUSABLE component (used for both set and rarity dropdowns).
2. Props interface:
   ```
   interface MultiSelectProps {
     label: string;           // "Set" or "Rarity"
     options: string[];       // list of option values
     selected: Set<string>;   // currently selected values
     onToggle: (value: string) => void;
     formatOption?: (value: string) => string;  // optional display formatter
   }
   ```
3. Internal state: `const [isOpen, setIsOpen] = useState(false)`
4. Button trigger:
   - Shows `label` text, and if `selected.size > 0`, appends count badge: `({selected.size})`
   - Classes: `rounded-md border border-zinc-300 dark:border-zinc-700 bg-transparent px-3 py-1.5 text-sm cursor-pointer hover:bg-zinc-50 dark:hover:bg-zinc-900`
   - Clicks toggles `isOpen`
5. Dropdown panel (only when isOpen):
   - Position: `absolute top-full mt-1 z-30 rounded-md border border-zinc-200 dark:border-zinc-700 bg-white dark:bg-zinc-900 shadow-lg`
   - Min width: `min-w-[200px]`
   - Max height with scroll: `max-h-60 overflow-y-auto`
   - Map over `options`, rendering each as a `<label>` with checkbox:
     - `<input type="checkbox" checked={selected.has(option)} onChange={() => onToggle(option)} />`
     - Display text: `formatOption ? formatOption(option) : option`
     - Classes for each row: `flex items-center gap-2 px-3 py-1.5 text-sm hover:bg-zinc-50 dark:hover:bg-zinc-800 cursor-pointer`
6. Close on outside click: when isOpen, render an invisible backdrop `<div className="fixed inset-0 z-20" onClick={() => setIsOpen(false)} />` behind the dropdown. This handles the "two dropdowns open at once" pitfall from research.
7. Wrap the entire component in `<div className="relative">` for positioning context.
