**Step 2: Create `src/db/queries.ts` (per D-04, D-05, D-06)**

Create the file with these exact exports:

1. `rowToCard(row: typeof cards.$inferSelect): Card` -- pure function, exported for testing
   - Maps Drizzle row to Card interface
   - Price conversion: `row.price !== null ? row.price / 100 : null` (D-05: cents to dollars)
   - Timestamp conversion: `row.createdAt.toISOString()`, `row.updatedAt.toISOString()`
   - All other fields pass through directly (Drizzle already returns camelCase property names from the schema definition -- no manual snake_case conversion needed)
   - Include `scryfallId: row.scryfallId` (D-07)

2. `getCards(): Promise<Card[]>` -- async function
   - Calls `db.select().from(cards)` then maps each row through `rowToCard`
   - Returns `Card[]` with prices in dollars

3. `getCardById(id: string): Promise<Card | null>` -- async function
   - Calls `db.select().from(cards).where(eq(cards.id, id)).limit(1)`
   - Returns `rowToCard(rows[0])` if found, `null` if empty
   - Import `eq` from `drizzle-orm`

4. `getCardsMeta(): Promise<CardData["meta"]>` -- async function (D-03)
   - Uses Drizzle aggregate: `db.select({ totalCards: count(), lastUpdated: max(cards.updatedAt) }).from(cards)`
   - Import `count` and `max` from `drizzle-orm`
   - Returns object with shape `{ lastUpdated: string, totalCards: number, totalSkipped: 0, totalMissingPrices: 0 }`
   - `lastUpdated`: `result.lastUpdated?.toISOString() ?? new Date().toISOString()`
   - `totalSkipped: 0` and `totalMissingPrices: 0` are constants (these meta fields are from the old CSV pipeline; CardGrid receives them but does not prominently display them)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DB client to Neon Postgres | Server-side only; DATABASE_URL must not leak to client |
| Server component to client component | Card data passed as props; no DB objects exposed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | src/db/queries.ts | mitigate | DATABASE_URL accessed only in server-side code; not prefixed with NEXT_PUBLIC_; db client.ts is server-only module; error catch blocks log to console.error (server logs) and return generic user message, never stack traces |
| T-07-02 | Tampering | src/db/queries.ts getCardById | mitigate | Uses Drizzle parameterized query via eq(cards.id, id) -- never raw string interpolation; prevents SQL injection |
| T-07-03 | Denial of Service | src/db/queries.ts getCards | accept | SELECT * from cards is bounded by inventory size (~136 cards); no user-controlled WHERE clauses on public pages; Neon serverless has built-in connection limits |
</threat_model>
