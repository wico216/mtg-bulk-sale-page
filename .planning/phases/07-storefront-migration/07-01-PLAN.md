Exports:

1. `rowToCard(row: typeof cards.$inferSelect): Card` -- pure function, exported for testing
   - Maps Drizzle row to Card interface
   - Price conversion: `row.price !== null ? row.price / 100 : null` (D-05: cents to dollars)
   - Timestamp conversion: `row.createdAt.toISOString()`, `row.updatedAt.toISOString()`
   - All other fields pass through directly (Drizzle already returns camelCase property names from the schema definition -- no manual snake_case conversion needed)
   - Include `scryfallId: row.scryfallId` (D-07)

2. `getCards(): Promise<Card[]>` -- async function
   - Calls `db.select().from(cards).orderBy(asc(cards.name))` then maps each row through `rowToCard`
   - Returns `Card[]` with prices in dollars
   - Deterministic ordering by name ASC provides stable results for client-side filtering (addresses review concern: unstable row order causing jitter)
   - NOTE: This fetches all cards unbounded. This is an explicit small-store assumption -- the inventory is ~136 cards. A pagination boundary would be added if inventory exceeds ~1000 cards in a future phase.

3. `getCardById(id: string): Promise<Card | null>` -- async function
   - Calls `db.select().from(cards).where(eq(cards.id, id)).limit(1)`
   - Returns `rowToCard(rows[0])` if found, `null` if empty
   - Import `eq` from `drizzle-orm`

4. `getCardsMeta(): Promise<CardData["meta"]>` -- async function (D-03)
   - Uses Drizzle aggregate: `db.select({ totalCards: count(), lastUpdated: max(cards.updatedAt) }).from(cards)`
   - Import `count` and `max` from `drizzle-orm`
   - Returns object matching `CardData["meta"]` EXACTLY:
     ```typescript
     return {
     ```
