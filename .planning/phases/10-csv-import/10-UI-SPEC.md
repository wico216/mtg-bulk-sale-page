| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge internal padding, inline row gaps |
| sm | 8px | Button/input padding-y (`py-2`), compact control spacing (`gap-2`) |
| sm+ | 12px | Action bar padding-y (`py-3`), table cell padding-y for headers, preview zone inner spacing (`gap-3`) |
| md | 16px | Container horizontal padding (`px-4`), card/panel internal padding (`p-4`), section gaps between preview zones |
| lg | 24px | Content area top padding (`pt-6`), gap between page heading and drop-zone / preview, button group `mb-6` |
| xl | 32px | Drop-zone vertical padding (`py-8`), gap between major stages (upload → progress → preview) |
| 2xl | 48px | Empty-state vertical padding (`py-12`), drop-zone tall mode on first render |

Exceptions:
- `sm+` (12px) is kept from Phase 9 because the admin header uses `py-3` and the action bar inherits it. Phase 10 reuses the same control (`<button>` sized `px-4 py-1.5` for the Import CSV button) so it aligns pixel-for-pixel with the existing Export CSV button.
- Drop-zone uses `py-8` (32px) + `px-4` (16px) for comfortable "drop target" affordance — already on-scale.

Source: Inherited from Phase 9 UI-SPEC (see `09-UI-SPEC.md §Spacing Scale`). Matches existing admin shell (`max-w-7xl mx-auto px-4 pt-6`).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption | 12px (`text-xs`) | 400 (regular) | 1.5 |
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 600 (semibold) | 1.5 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |

Font weights used: **400 (regular)** and **600 (semibold)**. No other weights.

Role usage for this phase:
- **Caption** (12px regular): Row numbers in skipped-rows list (`Row 47`), "Skipped rows (4)" expander count badge, progress subtext (`X / Y cards enriched`), file metadata (`filename.csv · 42 KB`).
- **Body** (14px regular): Drop-zone instruction text ("Drop a Manabox CSV here or click to browse"), sample card list rows (name, set, price, condition, quantity), skipped-rows list item text, cancel-button text, inline error banner body copy.
- **Label** (14px semibold): Preview zone headings ("Summary", "Sample (first 20)", "Skipped rows"), summary count numbers ("**143** cards will be imported"), "Confirm import — …" button text, "Cancel" button text, "Import CSV" action bar button, progress percentage.
- **Heading** (20px semibold): Page heading on `/admin/import` — `Import CSV`.

Source: Inherited from Phase 9 UI-SPEC. No new roles introduced. Monospace (Geist Mono via `--font-mono`) is reserved for set codes + collector numbers inside the skipped-rows list (e.g., `lea-232`) to improve scannability — uses the same 12px Caption size, regular weight.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` light / `#0a0a0a` dark | Page background, drop-zone default background, preview panel background, sample table background |
| Secondary (30%) | `zinc-50` (`#fafafa`) light / `zinc-900` (`#18181b`) dark | Sample list row stripes, skipped-rows expander background, progress-bar track background, disabled button background |
| Accent (10%) | `#4f46e5` (indigo-600) | "Import CSV" button (action bar), "Confirm import" button, drop-zone active/drag-over border + light-wash fill, progress-bar fill, sample-zone heading underline |
| Accent hover | `#4338ca` (indigo-700) | Hover state for accent-filled buttons |
| Accent light | `#eef2ff` (indigo-50) | Drop-zone drag-over background wash, progress-bar light track in light mode, "in-progress" soft banner background |
| Destructive | `#dc2626` (red-600) | "Confirm import — replace all X current cards …" button background (because import is destructive), inline error banner border + text, Scryfall-miss icon in skipped-rows list |
| Destructive light | `red-50` (`#fef2f2`) light / `red-950/20` dark | Inline error banner background, destructive-confirm button hover halo |
| Warning | `#d97706` (amber-600) | "Missing prices (Z)" summary number + warning icon (not fatal, but flags degraded data) |
| Success | `#16a34a` (green-600) | Post-commit toast icon (on `/admin` after redirect), "Parsed OK" checkmark next to filename once stream completes |

**Accent reserved for:**
1. "Import CSV" action-bar button (`/admin` page, next to Export CSV)
2. "Confirm import — replace N cards with M new cards" button on the preview screen *(the destructive semantic also applies — see note below)*
3. Drop-zone `drag-over` / hover border and background wash
4. Progress-bar filled portion
5. Nothing else. Cancel button and expander chevron use zinc-only styling.

**Destructive vs Accent on the Confirm button:** Per Phase 9 precedent (`DeleteConfirmation` uses `bg-red-600`), any button that deletes inventory uses the **destructive** color, not accent. Phase 10's "Confirm import" wipes the whole table in a single transaction (D-14), so the confirm button is **destructive red (`bg-red-600 hover:bg-red-700`)**, not accent indigo. The "Import CSV" entry button on `/admin` remains accent because the click merely navigates to `/admin/import` — non-destructive.

Source:
- `globals.css` defines `--color-accent: #4f46e5`, `--color-accent-hover: #4338ca`, `--color-accent-light: #eef2ff`.
- Destructive and success colors follow Phase 9 precedent (`src/app/admin/_components/delete-confirmation.tsx` uses `bg-red-600`; toast uses `bg-red-600`).
- Warning amber-600 matches cart stock-warning and Phase 9 low-stock badge pattern.

## Components & Visual Specs

### 1. Drop Zone (`src/app/admin/import/_components/drop-zone.tsx`)

**Purpose:** Click-to-browse + drag-drop file input for CSV upload.

**Visual (idle):**
- Container: `border-2 border-dashed border-zinc-300 dark:border-zinc-700 rounded-md p-8 text-center cursor-pointer transition-colors`
- Inner: `flex flex-col items-center gap-3`
- Icon: 48px document-arrow-up SVG (inline, Heroicons outline style, `stroke="currentColor" strokeWidth={1.5}`), `text-zinc-400 dark:text-zinc-500`
- Primary text: `text-sm text-zinc-700 dark:text-zinc-300` (Body)
- Secondary text: `text-xs text-zinc-500 dark:text-zinc-400 mt-1` (Caption)

**Visual (drag-over):**
- Container: `border-accent bg-accent-light dark:bg-indigo-950/20` (swap border + background)
- Icon: `text-accent`

**Visual (rejected / after invalid-extension alert):**
- Container: unchanged (remains idle-styled after dismissing the inline error banner)
- Inline error banner ABOVE the drop zone: `mb-3 bg-red-50 dark:bg-red-950/20 border border-red-200 dark:border-red-800 rounded-md p-3 text-sm text-red-600 dark:text-red-400` with copy `Only .csv files are supported. Please export a CSV from Manabox and try again.`

**Accessibility:**
- Container: `role="button" tabIndex={0}` with `onKeyDown` handling `Enter` and `Space`
- Hidden `<input type="file" accept=".csv">` triggered by click / keyboard activation
- `aria-label="Upload Manabox CSV file"`

**Interaction:**
- Click anywhere on container → opens file picker
- Drag-over → switches to drag-over visual state
- Drag-leave → reverts to idle
- Drop → validates extension, then triggers upload
- Enter/Space key → opens file picker (keyboard users)

### 2. Progress Bar (`src/app/admin/import/_components/progress-bar.tsx`)

**Purpose:** Live counter during NDJSON stream.

**Visual:**
- Wrapper: `w-full space-y-2`
- Track: `h-2 w-full rounded-full bg-zinc-100 dark:bg-zinc-800 overflow-hidden`
- Fill: `h-full bg-accent transition-[width] duration-150 ease-out` with inline `style={{ width: `${(done/total) * 100}%` }}`
- Counter below: `text-xs text-zinc-500 dark:text-zinc-400` reading `{done} / {total} cards enriched`

**ARIA:**
- Wrapper: `role="progressbar" aria-valuemin="0" aria-valuemax={total} aria-valuenow={done} aria-label="Enrichment progress"`
- `aria-live="polite"` on the counter text so screen readers announce milestone updates (every ~10% change, client-throttled)

**States:**
- Indeterminate (during initial parse, before total is known): fill uses `animate-pulse bg-zinc-300 dark:bg-zinc-700` at 25% width; counter reads `Reading your file…`
- Active: fill solid accent, width driven by `done/total`
- Complete: fill at 100% for ~300ms before transitioning to the preview panel (visual resolution)

## Responsive Behavior

Per REQUIREMENTS.md out-of-scope: "Mobile-optimized admin panel — Admin tasks are desktop workflows." No mobile breakpoints are designed.

- Drop-zone and preview panels are `max-w-3xl` (768px), which fits on a 768px+ viewport comfortably.
- On narrower widths, the sample table uses `overflow-x-auto` so it scrolls horizontally rather than reflowing.
- The delta label on the Confirm button wraps naturally to multiple lines at narrow widths — this is acceptable and preserves the safeguard.

Minimum comfortable width: 768px (md breakpoint). Below that, usability degrades gracefully; no dedicated mobile design.

## Accessibility Notes

- **Drop zone:** `role="button" tabIndex={0}`, keyboard-activatable via Enter/Space, `aria-label="Upload Manabox CSV file"`. Hidden file input retains native keyboard semantics via label association.
- **Progress bar:** `role="progressbar"` with `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, `aria-label="Enrichment progress"`. Counter text wrapped in `aria-live="polite"` (throttled updates, e.g., every 10% delta) so screen readers aren't spammed with 143 updates.
- **Preview zones:** Each zone uses a semantic heading (`<h2>` or `<h3>`) — not just a styled `<div>`. Summary, Sample, Skipped each get a heading so screen reader users can jump between zones.
- **Skipped-rows expander:** Use native `<details>` + `<summary>` for zero-JS keyboard support. The `<summary>` is focusable and Enter-activatable by default.
- **Sample table:** Semantic `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th>` (with `scope="col"`), `<td>`.
- **Confirm button:** `aria-describedby` pointing to the summary primary count, so screen readers hear both the button label AND the delta context.
- **Error banner:** `role="alert"` so it's announced when it appears.
- **Success toast:** `role="status"` (polite) — non-critical, success info.
- **Inline error on preview (commit failed):** `role="alert"` — the admin needs to know immediately.
- **Focus management on redirect (D-15):** After `router.push("/admin")`, focus should land on the success toast (or, if screen-reader-visible, on the page heading "Inventory"). The toast's `role="status"` announces regardless.
- **Color-only signals avoided:** The missing-prices warning uses amber color PLUS a warning icon. Scryfall-miss rows use red color PLUS the reason text — never color alone.
