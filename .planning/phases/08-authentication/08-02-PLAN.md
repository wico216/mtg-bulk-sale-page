1. Create `src/components/google-sign-in-button.tsx` -- Client component for Google sign-in via form POST (NOT server action -- known bug per 08-RESEARCH.md Pitfall 1).

   CRITICAL review fix (HIGH): The button MUST be disabled until the CSRF token is loaded. This prevents a race condition where users click before the token fetch completes, causing a failed submission with empty CSRF token.

   The form MUST include `callbackUrl=/admin` as a hidden field so the post-login redirect is deterministic (per D-03, review suggestion from Codex).

   ```typescript
   "use client";

   import { useState, useEffect } from "react";

   export function GoogleSignInButton() {
     const [csrfToken, setCsrfToken] = useState("");
     const [isLoading, setIsLoading] = useState(false);

     useEffect(() => {
       fetch("/api/auth/csrf")
         .then((res) => res.json())
         .then((data) => setCsrfToken(data.csrfToken))
         .catch(() => {
           // CSRF fetch failed -- button stays disabled
           // User can reload page to retry
         });
     }, []);

     return (
       <form
         method="post"
         action="/api/auth/signin/google"
         onSubmit={() => setIsLoading(true)}
       >
         <input type="hidden" name="csrfToken" value={csrfToken} />
         <input type="hidden" name="callbackUrl" value="/admin" />
         <button
           type="submit"
           disabled={!csrfToken || isLoading}
           className="w-full flex items-center justify-center gap-2 px-5 py-3 text-sm font-bold rounded-md bg-accent text-white hover:bg-accent-hover transition-colors focus:ring-2 focus:ring-accent/50 focus:ring-offset-2 disabled:opacity-70 disabled:cursor-not-allowed"
         >
           {/* Inline Google "G" SVG icon, 20x20 */}
   ```

|----------|-------------|
| Browser -> Login page | Public page, renders for all visitors; sign-in form POSTs to Auth.js endpoint with CSRF token |
| Auth.js callback -> admin pages | OAuth response is validated by Auth.js before session is created |
| Session cookie -> admin layout | Layout reads session for display; proxy.ts + page auth() provide defense-in-depth |
| Browser -> /api/admin/* | Protected by requireAdmin() in route handlers; proxy passes through (no redirect) |
| CSRF token -> sign-in form | Token fetched async; button disabled until loaded to prevent submission without token |

| T-08-09 | Spoofing | Login page CSRF | mitigate | CSRF token fetched from `/api/auth/csrf` and included in sign-in form POST; button disabled until token loaded (prevents race condition); Auth.js validates token server-side |
