3. Create `src/proxy.ts` -- Route protection (per D-03, D-06, D-07). Must be at `src/` root, same level as `src/app/`. Exports `default` which is the `auth()` wrapper. Matcher covers `/admin/:path*` and `/api/admin/:path*`:
   ```typescript
   import { auth } from "./auth";
   import { NextResponse } from "next/server";

   export default auth((req) => {
     const { pathname } = req.nextUrl;
     const isAdminRoute = pathname.startsWith("/admin");
     const isAdminApi = pathname.startsWith("/api/admin");
     const isLoginPage = pathname === "/admin/login";
     const isAccessDenied = pathname === "/admin/access-denied";
     const isAuthApi = pathname.startsWith("/api/auth");

     // Never block auth API routes
     if (isAuthApi) return NextResponse.next();

     // Admin page routes need session check
     if (isAdminRoute) {
       if (!req.auth) {
         // Unauthenticated -> login page (D-07)
         if (isLoginPage || isAccessDenied) return NextResponse.next();
         return NextResponse.redirect(new URL("/admin/login", req.url));
       }

       const isAdmin = req.auth.user?.email === process.env.ADMIN_EMAIL;

       // Authenticated admin on login page -> redirect to admin dashboard (D-03)
       if (isLoginPage && isAdmin) {
         return NextResponse.redirect(new URL("/admin", req.url));
       }

       // Authenticated non-admin -> access denied (D-06)
       if (!isAdmin && !isAccessDenied && !isLoginPage) {
         return NextResponse.redirect(new URL("/admin/access-denied", req.url));
       }
     }

     // Admin API routes: proxy only blocks unauthenticated (authoritative check in route handler)
     // API routes handle their own 401/403 via requireAdmin() -- proxy is optimistic only

     return NextResponse.next();
   });

   export const config = {
     matcher: ["/admin/:path*", "/api/admin/:path*"],
   };
   ```

1. Create `src/lib/auth/admin-check.ts` (per D-05, D-08):
   ```typescript
   import "server-only";
   import { auth } from "@/auth";

   export type AdminSession = {
     user: { email: string; name: string; image?: string };
   };

   /**
    * Verifies the request has a valid admin session.
    * Returns the session if admin, or a Response with 401/403 error per D-08.
    */
   export async function requireAdmin(): Promise<AdminSession | Response> {
     const session = await auth();

     if (!session?.user?.email) {
       return Response.json({ error: "Unauthorized" }, { status: 401 });
     }

     if (session.user.email !== process.env.ADMIN_EMAIL) {
       return Response.json({ error: "Forbidden" }, { status: 403 });
     }

     return session as AdminSession;
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> proxy.ts | Untrusted requests hit proxy before any route code; session cookie is the trust signal |
| proxy.ts -> Server Components | proxy.ts is optimistic only; server components must re-verify auth |
| Browser -> /api/admin/* | API requests from client carry session cookie; requireAdmin() is authoritative gate |
| Google OAuth -> Auth.js | OAuth callback carries authorization code; Auth.js validates state and exchanges code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | OAuth callback | mitigate | Auth.js handles OAuth state parameter and PKCE automatically; no hand-rolled OAuth |
| T-08-02 | Tampering | JWT session cookie | mitigate | Auth.js signs JWT with AUTH_SECRET (HS256); HttpOnly + Secure + SameSite=lax cookie flags set by Auth.js |
| T-08-03 | Elevation of Privilege | Non-admin accessing /admin | mitigate | Three-layer defense: proxy.ts redirect (optimistic), auth() in server components (authoritative), requireAdmin() in API routes (authoritative). Email compared server-side against ADMIN_EMAIL env var |
| T-08-04 | Spoofing | CSRF on sign-in/sign-out | mitigate | Auth.js generates and validates CSRF tokens on all POST endpoints automatically |
| T-08-05 | Spoofing | Open redirect via callbackUrl | mitigate | Auth.js validates redirect URLs against allowed origins by default |
| T-08-06 | Information Disclosure | AUTH_SECRET / ADMIN_EMAIL exposure | mitigate | Env vars in .env.local (gitignored), never embedded in client bundles; admin-check.ts uses "server-only" import guard |
| T-08-07 | Denial of Service | Brute force OAuth attempts | accept | Low risk -- Google OAuth rate-limits sign-in attempts; friend store with no public user base |
| T-08-08 | Information Disclosure | Session cookie theft via XSS | mitigate | HttpOnly cookie flag prevents JS access; Next.js CSP headers recommended for production (out of scope for this phase but noted) |
</threat_model>
