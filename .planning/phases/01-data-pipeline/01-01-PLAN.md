<task type="auto">
  <name>Task 2: Define card data model types</name>
  <files>src/lib/types.ts</files>
  <action>
    Create `src/lib/types.ts` with the following type definitions exactly matching the research specification:

    1. `ManaboxRow` interface -- maps to CSV headers with PapaParse header mode:
       - `Name`: string
       - `Set code`: string
       - `Set name`: string
       - `Collector number`: string
       - `Foil`: string (values: "foil" | "normal")
       - `Rarity`: string (values: "common" | "uncommon" | "rare" | "mythic")
       - `Quantity`: number (PapaParse dynamicTyping converts this)
       - `Condition`: string (values like "near_mint", "lightly_played", etc.)
       - Plus remaining CSV fields as optional strings (ManaBox ID, Scryfall ID, etc.) so PapaParse doesn't complain, but these are ignored in processing

    2. `Card` interface -- the enriched card model:
       - `id`: string (composite key: `${setCode}-${collectorNumber}-${foil}-${condition}`)
       - `name`: string
       - `setCode`: string (lowercased)
       - `setName`: string
       - `collectorNumber`: string
       - `price`: number | null (null means "Price N/A")
       - `condition`: string
       - `quantity`: number
       - `colorIdentity`: string[] (from Scryfall, e.g. ["G"], ["W","U"])
       - `imageUrl`: string | null (null if no image available)
       - `rarity`: string
       - `foil`: boolean

    3. `CardData` interface -- the generated JSON structure:
       - `cards`: Card[]
       - `meta`: object with `lastUpdated` (string), `totalCards` (number), `totalSkipped` (number), `totalMissingPrices` (number)

    4. `ScryfallCard` interface -- key fields from Scryfall API response (used in enrichment):
       - `object`: string
       - `name`: string
       - `color_identity`: string[]
       - `image_uris?`: { normal: string; small: string; large: string }
       - `card_faces?`: Array<{ name: string; image_uris?: { normal: string } }>
       - `prices`: { usd: string | null; usd_foil: string | null; usd_etched: string | null }
       - `layout`: string

    Export all four interfaces.
  </action>
  <verify>`npx tsc --noEmit src/lib/types.ts` compiles without errors.</verify>
  <done>All four interfaces (ManaboxRow, Card, CardData, ScryfallCard) are defined and exported with correct field types matching CSV structure and Scryfall API response.</done>
</task>

<task type="auto">
  <name>Task 3: Implement CSV parser with PapaParse and duplicate merging</name>
  <files>src/lib/csv-parser.ts</files>
  <action>
    Create `src/lib/csv-parser.ts` that:

    1. Exports `parseAllCsvFiles(inventoryDir: string): Card[]` function that:
       a. Uses fast-glob to find all `*.csv` files in the given directory
       b. For each CSV file, calls `parseSingleCsv(filePath)` 
       c. Merges all results and deduplicates

    2. Internal `parseSingleCsv(filePath: string): Card[]` function that:
       a. Reads file with `readFileSync(filePath, 'utf-8')`
       b. Parses with `Papa.parse<ManaboxRow>(content, { header: true, dynamicTyping: true, skipEmptyLines: true })`
       c. Logs any parse errors as warnings (do not throw)
       d. Maps each ManaboxRow to a partial Card (without Scryfall fields):
          - `id`: `${setCode}-${collectorNumber}-${foil ? 'foil' : 'normal'}-${condition}`
          - `setCode`: lowercased from CSV `Set code`
          - `foil`: `row.Foil === 'foil'`
          - `price`: null (filled by enrichment later)
          - `colorIdentity`: [] (filled by enrichment later)
          - `imageUrl`: null (filled by enrichment later)
          - All other fields mapped from CSV columns
       e. Skips rows where Name, Set code, or Collector number is missing/empty

    3. Internal `mergeCards(cards: Card[]): Card[]` that:
       a. Groups cards by `id` (the composite dedup key)
       b. For duplicates, sums `quantity` and keeps first occurrence for other fields
       c. Returns merged array

    The function signature uses Card[] as return type, but price/colorIdentity/imageUrl will be null/empty until enrichment fills them in. This is intentional -- the CSV parser produces "partial" cards that enrichment completes.

    Import types from `./types`. Use `import Papa from 'papaparse'` and `import { globSync } from 'fast-glob'`.
  </action>
  <verify>
    Create a quick test: `npx tsx -e "const { parseAllCsvFiles } = require('./src/lib/csv-parser'); const cards = parseAllCsvFiles('data/inventory'); console.log(cards.length, 'cards parsed'); console.log(cards[0]);"` -- should output the number of cards and first card object with null price/imageUrl/empty colorIdentity.

    If the above import style doesn't work with tsx, use: `npx tsx -e "import { parseAllCsvFiles } from './src/lib/csv-parser'; const cards = parseAllCsvFiles('data/inventory'); console.log(cards.length, 'cards parsed'); console.log(JSON.stringify(cards[0], null, 2));"` 
  </verify>
  <done>parseAllCsvFiles reads all CSVs from data/inventory/, maps Manabox fields to Card type, merges duplicates by summing quantities, and returns Card[] with null enrichment fields. Running against "Blue Binder.csv" returns ~135 cards.</done>
