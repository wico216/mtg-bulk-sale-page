<task type="auto">
  <name>Task 1: Implement file-based cache and rate-limited Scryfall client</name>
  <files>
    src/lib/cache.ts
    src/lib/scryfall.ts
  </files>
  <action>
    **src/lib/cache.ts:**
    Create a file-based JSON cache with 24-hour TTL (per user decision on data freshness).

    1. Export `getCached<T>(key: string): T | null` that:
       - Builds path: `data/cache/scryfall/${sanitizedKey}.json`
       - Returns null if file doesn't exist
       - Reads and parses the JSON file as `{ timestamp: number, data: T }`
       - Returns null if `Date.now() - timestamp > 24 * 60 * 60 * 1000` (expired)
       - Returns `data` if valid

    2. Export `setCache<T>(key: string, data: T): void` that:
       - Creates `data/cache/scryfall/` directory recursively if needed
       - Writes `{ timestamp: Date.now(), data }` as JSON to the cache file

    3. Internal `sanitizeKey(key: string): string` that replaces filesystem-unsafe characters (/, \, :, *, ?, ", <, >, |, and any non-alphanumeric except hyphen) with underscores. This handles collector numbers like "123a" or special characters.

    **src/lib/scryfall.ts:**
    Create a rate-limited Scryfall API client.

    1. Module-level `lastRequestTime = 0` variable for rate limiting

    2. Internal `rateLimitedFetch(url: string): Promise<Response>` that:
       - Calculates elapsed time since last request
       - If less than 100ms elapsed, awaits the difference via setTimeout
       - Updates lastRequestTime
       - Calls and returns `fetch(url)`

    3. Export `fetchCard(setCode: string, collectorNumber: string): Promise<ScryfallCard | null>` that:
       - Builds cache key: `${setCode}-${collectorNumber}`
       - Checks cache first via `getCached<ScryfallCard>(cacheKey)`
       - If cached, returns cached data (log "cache hit" at debug level)
       - If not cached, calls `rateLimitedFetch(`https://api.scryfall.com/cards/${setCode.toLowerCase()}/${collectorNumber}`)`
       - On 404: returns null (card not found)
       - On other non-OK status: logs warning, returns null (do not throw -- graceful degradation)
       - On success: parses JSON, calls `setCache(cacheKey, data)`, returns data

    Import ScryfallCard from `./types`. Import getCached/setCache from `./cache`.

    Per user decision: match by set code + collector number. Per research: SLD high collector numbers resolve correctly via standard endpoint, so the name+set fallback requested in CONTEXT.md is NOT needed (research verified 1750, 2071, 7028 all resolve). Still, implement the standard endpoint and let the "skip if not found" logic handle any edge cases.
  </action>
  <verify>

    Quick integration test: `npx tsx -e "import { fetchCard } from './src/lib/scryfall'; fetchCard('sld', '1750').then(c => console.log(c?.name, c?.prices?.usd));"` should print "Heroic Intervention" and a price string.
    
    Run again immediately to verify cache hit (should be near-instant, no 100ms delay).
  </verify>
  <done>Cache reads/writes JSON files with 24h TTL. Scryfall client fetches cards with 100ms rate limiting, caches responses, returns null for missing cards. Verified with real API call to SLD/1750.</done>

<task type="auto">
  <name>Task 2: Implement card enrichment pipeline</name>
  <files>src/lib/enrichment.ts</files>
  <action>
    Create `src/lib/enrichment.ts` that orchestrates enriching parsed cards with Scryfall data.

    1. Internal `getImageUrl(card: ScryfallCard): string | null` that:
       - Returns `card.image_uris.normal` if `image_uris` exists
       - Falls back to `card.card_faces?.[0]?.image_uris?.normal` for double-faced cards
       - Returns null if neither exists
       (Per research pitfall 1: double-faced cards have no top-level image_uris)

    2. Internal `getPrice(prices: ScryfallCard['prices']): number | null` that:
       - Tries `prices.usd` first (primary -- per research, this works for both foil and non-foil)
       - Falls back to `prices.usd_foil`
       - Falls back to `prices.usd_etched`
       - Returns null if all are null/undefined
       - Parses the string to number via `parseFloat()` (Scryfall returns prices as strings like "16.05")
       (Per research pitfall 2 and user decision: usd first, then usd_foil, null triggers "Price N/A")

    3. Export `enrichCards(cards: Card[]): Promise<{ cards: Card[], stats: { processed: number, skipped: number, missingPrices: number } }>` that:
       - Iterates over cards sequentially (NOT in parallel -- rate limiting is sequential)
       - For each card, calls `fetchCard(card.setCode, card.collectorNumber)`
       - If Scryfall returns null: increment `skipped`, DO NOT include card in output (per user decision: skip cards not found on Scryfall)
       - If Scryfall returns data: fill in enrichment fields:
         - `card.imageUrl` = `getImageUrl(scryfallData)`
         - `card.price` = `getPrice(scryfallData.prices)`
         - `card.colorIdentity` = `scryfallData.color_identity`
         - If price is null, increment `missingPrices` (card still included per user decision)
       - Logs progress every 25 cards: "Enriching: X/Y cards processed..."
       - Returns enriched cards array and stats object

    Import Card and ScryfallCard from `./types`. Import fetchCard from `./scryfall`.

    The function mutates the card objects in place for simplicity (they're already copies from CSV parsing). Cards that fail Scryfall lookup are excluded from the returned array entirely.
  </action>

  </verify>
