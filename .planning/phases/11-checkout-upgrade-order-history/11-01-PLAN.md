<objective>
Make checkout real: placing an order atomically decrements stock and persists an order history record. This plan does not build admin order pages yet; it establishes the transactional write path and updates the buyer checkout UX for stale-cart conflicts.

Core invariant: inventory is never oversold. The atomic database write, not the email send, defines a placed order.
</objective>

    Add `src/db/orders.ts` with the `placeCheckoutOrder` helper.

    Implementation shape:
    - Import `sql` from `drizzle-orm` and execute the write through `db.execute(...)` with a Drizzle SQL template.
    - Use Drizzle `sql` template values, not string concatenation.
    - Build a `requested(card_id, requested_qty)` VALUES table from aggregated inputs. Use `sql.join(...)` or an equivalent parameterized Drizzle SQL composition helper; never interpolate raw card IDs or quantities into SQL text.

    - Return a single JSON payload containing either inserted order data or conflicts.
