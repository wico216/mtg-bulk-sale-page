# Phase 10.1 Plan 02: DELETE /api/admin/inventory Summary

Destructive admin endpoint that atomically wipes the cards table and returns the pre-delete count, gated by `requireAdmin()`, reusing the existing `replaceAllCards([])` helper — no new query plumbing, no `db.transaction()`.

## Route

`DELETE /api/admin/inventory`

| Status | Body                                                       | When                                                                          |
| ------ | ---------------------------------------------------------- | ----------------------------------------------------------------------------- |
| 200    | `{ success: true, deleted: <previousTotal> }`              | requireAdmin OK; getCardsMeta + replaceAllCards([]) both succeed              |
| 200    | `{ success: true, deleted: 0 }`                            | inventory was already empty; replaceAllCards([]) is still invoked exactly once (defense in depth — UI hides the button at total=0 but the route stays safe) |
| 401    | `{ error: "Unauthorized" }`                                | requireAdmin returns 401; replaceAllCards is NOT called                       |
| 403    | `{ error: "Forbidden" }`                                   | requireAdmin returns 403; replaceAllCards is NOT called                       |
| 500    | `{ error: "Delete failed — inventory unchanged" }`         | replaceAllCards rejects (DB error); literal em dash U+2014 to match /commit   |

`runtime = "nodejs"` and `maxDuration = 30` — same as `/api/admin/import/commit`. DB-only path: one `count()` + one `db.batch([db.delete(cards)])`.

## Atomicity & driver constraints

The handler delegates atomicity to the existing `replaceAllCards([])` helper from Phase 10-01:

```typescript
// src/db/queries.ts (existing — landed 10-01)
if (newCards.length === 0) {
  await db.batch([db.delete(cards)]);
  return { inserted: 0 };
}
```

`db.batch()` routes through Neon's HTTP `transaction()` endpoint and is atomic end-to-end on `drizzle-orm/neon-http`. We do NOT introduce `db.transaction()` — that interface throws "No transactions support in neon-http driver" (Phase 10 RESEARCH Pitfall 1). Confirmed via `grep -F "db.transaction(" src/app/api/admin/inventory/route.ts` → 0 matches.

## Read-before-wipe sequencing

`replaceAllCards([])` returns `{ inserted: 0 }` for the empty-array path — by definition, it cannot report how many rows were destroyed. So the response's `deleted: N` count comes from a `getCardsMeta()` read placed BEFORE the destructive call:

```typescript
const meta = await getCardsMeta();
const previousTotal = meta.totalCards;
await replaceAllCards([]);
return Response.json({ success: true, deleted: previousTotal });
```

Test E pins this ordering with `mock.invocationCallOrder` so a future refactor can't accidentally swap the calls and start returning `deleted: 0` for non-empty wipes.
