# Phase 10.1 Plan 05: Admin UI Delete-All Modal Summary

Inline-first destructive UX on `/admin`: a red "Delete all inventory" button slots into the existing ActionBar after Export CSV, opens a typed-DELETE confirmation modal, calls `DELETE /api/admin/inventory` (Wave 1), and surfaces a green "Deleted all N cards" toast inline before the table re-renders to empty state.

Three surgical edits + one new file. No tests added (manual verification deferred to 10.1-06; Wave 1 already pinned the route's behavior in `delete.test.ts`).

## Behavior

| Scenario | Outcome |
| -------- | ------- |
| `currentTotal === 0` | Delete-all button NOT rendered (hidden, not disabled) — D-11 |
| `currentTotal > 0`, click Delete-all | DeleteInventoryModal opens; input autofocused |
| Type partial "DEL" / "delete" / "DELETe" | Confirm button stays disabled (case-sensitive trimmed equality) — D-12 |
| Type exactly "DELETE" | Confirm button becomes enabled |
| Click Confirm | `fetch DELETE /api/admin/inventory` → `{ success: true, deleted: N }` → bubbles to InventoryTable |
| Successful delete | Modal closes; green toast "Deleted all N cards" appears inline (4s); table refetches and shows empty state; Delete-all button auto-hides — D-14 |
| 4xx/5xx response | Modal stays open with inline red banner "Delete failed — your inventory was not changed. {server message}"; user can retry or Cancel |
| Network error | Same inline banner; "Network error" or `err.message` |
| Press Escape | Modal closes (only when `!deleting`) |
| Click backdrop | Modal closes (only when `!deleting`); inner card clicks do NOT close |

## Deviations from Plan

### Decision deviation: toast handoff is parent-state, not sessionStorage

CONTEXT.md and the plan's `must_haves.truths` block referenced "reusing sessionStorage admin-toast handoff". The PLAN.md's `<interfaces>` block then proposed (and approved) a deviation: since both producer (DeleteInventoryModal) and consumer (InventoryTable) live on `/admin` without any navigation, a parent-state callback (`onSuccess(deletedCount)`) is cleaner than a cross-route sessionStorage hop. The sessionStorage `admin-toast` path remains the **import-flow** handoff (Phase 10-03 D-15) and is preserved verbatim — it's still read on InventoryTable mount.

This deviation matches the plan's own approved interface revision (PLAN.md line 134: "Decision: bubble onSuccess to InventoryTable, set toast inline, call router.refresh()…"). The user-visible D-14 toast behavior is identical: green "Deleted all N cards" for ~4s.

### No other deviations

The four task action blocks were implemented byte-for-byte against the plan's prescribed code:

- DeleteInventoryModal — pasted exactly as written; one-character ellipsis "…" preserved on the "Deleting…" button label
- ActionBar — pasted exactly as written; the Phase 9 D-12 / D-09 / D-11 comment annotations preserved
- InventoryTable — three surgical edits per Steps A/B/C/D; existing sessionStorage admin-toast useEffect untouched

## Notes for downstream plan 10.1-06 (manual verification)

Verification matrix the human should walk through on the Vercel preview URL:

1. **Visibility (D-11)**: when inventory is empty, the Delete-all button must NOT be in the DOM. Inspect `<ActionBar>` markup directly.
2. **Typed-DELETE safeguard (D-12)**:
   - Empty input → Confirm disabled
   - "DEL" → Confirm disabled
   - "delete" (lowercase) → Confirm disabled
   - " DELETE " (with surrounding whitespace) → Confirm enabled (trim allowed)
   - "DELETE" → Confirm enabled
3. **Happy path**: ~5-card inventory → click Delete-all → type DELETE → Confirm. Expect: modal closes, green "Deleted all 5 cards" toast appears for ~4s, table re-renders to empty state, Delete-all button vanishes.
4. **Cancel / Escape / backdrop**: all three close the modal without firing the DELETE.
5. **Cancel disabled while deleting**: artificially slow the network (DevTools → Slow 3G) and verify the Cancel + backdrop + Escape are inert during the in-flight DELETE; the input is also disabled.
6. **Error path**: temporarily break the route (rename to `/api/admin/inventory.bak` on the server, or DevTools-block the URL) → click Confirm. Expect: modal stays open with the red "Delete failed — your inventory was not changed." banner.
7. **Cart reconciliation (D-15 from CONTEXT)**: pre-populate cart → wipe inventory → visit `/cart`. Items should silently disappear (Phase 10 D-13 reconciliation already covers this — no extra work from this plan).
