## Decisions Made

- **`mergeCards` reuse, not duplication.** The plan called this out explicitly and it played out cleanly: concatenate the per-file `cards[]` into one array, then call `mergeCards(allCards)` exactly once. No new merge implementation, no new dedup tests beyond verifying the cross-file effect — the `Map<id, Card>` semantics were already covered by Phase 10-01 within-file tests.
- **`SkippedRow.filename` stays optional.** Forcing it would cascade through every consumer (UI, contract, single-file path). Optional + only-set-by-multi-file keeps the change surgical and backward-compatible. Test E asserts the single-file path explicitly leaves it `undefined`.
- **Per-file row numbers, not global.** Each file's header is row 1 inside that file (matches what an admin sees if they open `Blue Binder.csv` in Excel). The plan baked this into Test B's `rowNumber === 2` assertion.
- **Contract addition is type-only.** No production code in `/api/admin/import/preview` or `/api/admin/import/commit` references the new `filename` field yet — that's 10.1-02's job. This plan ships the type-level surface without route changes.
