# Phase 6 -- UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification: BACKEND-ONLY

Phase 6 (Database Foundation) has **no user-facing UI changes**. All work is infrastructure:

- Neon Postgres database provisioning
- Drizzle ORM schema definitions (cards, orders, order_items)
- Data migration from static JSON to database via seed script

The storefront continues reading static JSON during this phase (decision D-10). No admin panel exists yet (Phase 8+). No new pages, components, or visual changes are introduced.

**Impact on existing UI:** None. The storefront renders identically before and after Phase 6.

## Notes for Future Phases

The following phases will require full UI-SPEC contracts:

- **Phase 8 (Authentication):** Login page, admin layout shell, protected route indicators
- **Phase 9 (Admin Inventory Management):** Sortable table, inline editing, delete confirmation, low-stock highlight
- **Phase 10 (CSV Import):** File upload, preview table, import confirmation dialog
- **Phase 11 (Checkout Upgrade & Order History):** Order history table, order detail view, stock error states
- **Phase 12 (Bulk Operations & Dashboard):** Checkbox selection, bulk delete confirmation, stat cards, chart breakdowns

When those phases are reached, a shadcn initialization decision should be revisited. The existing v1.0 storefront uses custom components with Tailwind CSS v4, but the admin panel (Phases 8-12) would benefit from a component library for tables, dialogs, forms, and data display patterns.
