# Phase 10.1 Plan 01: Multi-CSV Import + Delete Inventory

## Objective

Add the two requested insertion features before Phase 11:

1. Admin can upload multiple Manabox CSV files in one import preview. The import remains a full-replace operation, not an incremental merge against existing inventory.
2. Admin can delete the entire inventory from `/admin` with an explicit inline destructive confirmation.

## Scope

### In scope

- Multi-file drag/drop and file picker support on `/admin/import`.
- `POST /api/admin/import/preview` accepts repeated multipart `file` fields.
- CSV parser can parse multiple uploaded contents and merge duplicate card IDs across files by summing quantities.
- Preview shows source-file parse counts and file names for parse skips.
- `DELETE /api/admin/cards` auth-gated endpoint deletes all cards and returns deleted row count.
- `/admin` action bar shows a `Delete inventory` button, disabled when the full inventory is empty.
- Inline confirmation warns that storefront will be empty until a new import.
- Tests cover multi-file parsing, multi-file preview route payload, delete-all route, and delete-all query helper.

### Out of scope

- Incremental or differential import against existing DB rows.
- Undo/version history after delete.
- Bulk selected-row deletion (Phase 12 remains the owner for selected bulk operations).
- Order history or stock decrement (Phase 11).

## Implementation notes

- Preserve the existing full-replace import contract: preview still buffers full enriched `cards[]`, and commit still calls `replaceAllCards(cards)`.
- Keep one multipart field constant (`IMPORT_FILE_FIELD = "file"`), but append it once per selected CSV.
- Preserve row numbers per source CSV, and add `fileName` only when multiple-upload context is available.
- Use the existing admin Route Handler auth pattern (`requireAdmin()`).
- Delete-all uses a dedicated `deleteAllCards()` helper rather than overloading `replaceAllCards([])` so the API can report deleted count.
