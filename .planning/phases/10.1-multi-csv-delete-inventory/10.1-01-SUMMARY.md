# Phase 10.1 Plan 01 Summary

Implemented the Phase 10.1 insertion before Phase 11: multi-CSV full-replace import and a delete-all inventory control.

## What changed

### Multi-CSV import

- `DropZone` now accepts multiple `.csv` files by drag/drop or file picker.
- `ImportClient` appends each file under the shared `IMPORT_FILE_FIELD` multipart key and displays multi-file upload/preview copy.
- `POST /api/admin/import/preview` now reads `formData.getAll(IMPORT_FILE_FIELD)`, validates every upload as a CSV file, parses them as one batch, and streams the existing NDJSON progress/result contract.
- `parseManaboxCsvContents()` was added to parse multiple CSV contents, preserve per-file row numbers for skipped rows, and merge duplicate card IDs across files by summing quantities.
- `PreviewPayload` now includes `sourceFiles`, and parse skipped rows can include `fileName`.
- `PreviewPanel` shows source-file parse counts when more than one CSV was uploaded and prefixes parse skip rows with the source filename.

### Delete inventory

- `deleteAllCards()` was added to `src/db/queries.ts`; it deletes all card rows and returns the deleted count.
- `DELETE /api/admin/cards` was added with the existing `requireAdmin()` route protection and a clear failure response.
- `ActionBar` now includes a red-outline `Delete inventory` button, disabled when the full inventory is empty.
- `InventoryTable` now tracks full inventory count separately from filtered table count, renders an inline destructive confirmation, clears local table/filter state after success, and shows success/error toasts in both populated and empty states.

### Admin login fallback

- Added Auth.js Credentials provider alongside optional Google OAuth because Google rejected the automation/browser context during local verification.
- The credentials provider is local-only: enabled outside production, disabled whenever `NODE_ENV=production`, and optionally hidden locally with `ENABLE_PASSWORD_LOGIN=false`.
- `ADMIN_USERNAME` and `ADMIN_PASSWORD` are env-backed secrets; no credentials are stored in source.
- Credentials authorization returns the `ADMIN_EMAIL` identity, so the existing proxy/admin/API authorization gate remains the single source of admin access truth.
- Added a credentials form to `/admin/login`; Google sign-in still renders when `AUTH_GOOGLE_ID` and `AUTH_GOOGLE_SECRET` are configured.
