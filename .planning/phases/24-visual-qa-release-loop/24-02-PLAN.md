# Plan 24-02 — Mobile Storefront Proof Exemplar

**Phase:** 24 — Visual QA Release Loop
**Risk:** Medium
**Goal:** Add the first real reference gate for the new loop: a mobile storefront UI Review packet that demonstrates proof, checklist, approval/fail, and regression-learning expectations.

## Outcome

`/qa/gates` includes a named Visual QA Gate that shows Wiko exactly how future mobile/UI changes should be reviewed from a phone or browser-openable Telegram/Vercel URL.

## Scope

### In scope

- Add a gate run such as `mobile-storefront-visual-qa-loop`.
- Checklist should cover:
  - mobile viewport proof is present
  - storefront header/search/filter/sort remain usable
  - no horizontal overflow at 390px/375px
  - proof is remote-reviewable, not local-only
  - failed review notes tell Atlas what to fix
- Link to deployment/preview artifact placeholders that are clearly marked until a live branch proof exists.
- Add Playwright coverage that opens the new gate after QA login and validates the key sections/checklist rows.
- Keep the gate framed as **Visual QA / UI Review / UAT sign-off**, not only the internal phrase “Playwright Approval Gates.”

### Out of scope

- Fixing a new mobile UI bug unless execution discovers a concrete issue.
- Production deployment.
- Uploading new real videos unless the implementation slice includes proof capture.

## Candidate files

- `src/lib/qa-gates.ts` / new gate registry module from 24-01
- `e2e/qa-gates.spec.ts`
- `docs/qa-approval-gates.md`

## Implementation steps

1. Add the mobile storefront reference gate metadata.
2. Make artifact labels honest: preview/deployment URL, screenshot/video placeholder, or actual proof URL if generated.
3. Add/extend Playwright test:
   - login through `/qa/login`
   - open `/qa/gates/mobile-storefront-visual-qa-loop`
   - assert heading, proof section, expected behavior, checklist rows, notes, approve disabled until required pass
4. If actual proof capture is feasible in this slice, record a mobile screenshot/video from local fixture or preview and attach it to the gate. Otherwise keep this as a reference packet and make the lack of live proof explicit.
5. Update docs to show this gate as the canonical reference loop.
