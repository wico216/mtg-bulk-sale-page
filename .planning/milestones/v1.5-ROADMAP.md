# v1.5 Roadmap — Visual QA Release Loop

**Status:** Planned / GSD kickoff 2026-06-25
**Theme:** Turn Spellbook's QA approval gate into a repeatable loop-engineering release system.

## Why this milestone exists

Spellbook already has the core pieces of a human-in-the-loop approval surface:

- `/qa/gates` and `/qa/gates/[runId]`
- password-protected QA login via `QA_GATE_PASSWORD`
- persisted `admin_audit_log` rows using `action = 'qa_gate.review'`
- server-side review enforcement for required checklist rows and failed-gate notes
- Playwright coverage in `e2e/qa-gates.spec.ts`

The current gap is repeatability. Gate packets are still mostly hand-authored in `src/lib/qa-gates.ts`, proof artifacts are manually referenced, and release discipline depends on Atlas remembering the process.

v1.5 makes the process explicit and reusable: every AI-generated UI/product change can produce proof, present a browser-openable QA gate to Wiko, block release when failed/unreviewed, and leave behind better tests/runbooks.

## Success definition

A Spellbook change is considered release-ready only when:

1. The implementation branch has real test/proof output.
2. A QA gate packet exists with preview URL, proof artifact, checklist, and expected behavior.
3. Wiko can approve/fail remotely from `/qa/gates/[runId]`.
4. A release guard can report approved / failed / pending status before merge/release.
5. Failed gates generate actionable fix notes for the next agent loop.
6. The process is documented clearly enough to reuse for Nova/work UAT later.

## Non-goals

- No public signup or generalized QA SaaS.
- No replacement for GitHub/Vercel CI.
- No auto-merge on approval in the first iteration.
- No sensitive/customer data in proof artifacts.
- No dependency on local-only video paths for remote review.

## Safety constraints

- Keep the QA surface password-protected.
- Do not rely on UI-disabled controls for approval rules; server-side validation remains mandatory.
- Failed gates must require notes.
- Approval history stays auditable via `admin_audit_log`.
- Release automation must fail closed when the gate status cannot be read.
